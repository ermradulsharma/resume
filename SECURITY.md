
# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 0.2.x   | :white_check_mark: |

## Reporting a Vulnerability

We take the security of this project seriously. If you discover a security vulnerability, please report it to us immediately.

### How to Report

Please **DO NOT** create a public GitHub issue for security vulnerabilities. Instead, please report them via email to:

**mradulsharma786@gmail.com**

Please include the following information in your report:

- Type of vulnerability (e.g., XSS, SQL Injection, RCE, etc.)
- Full details of the vulnerability, including step-by-step instructions to reproduce it.
- Proof of Concept (PoC) code or screenshots, if available.
- Impact of the vulnerability.

### What to Expect

- We will acknowledge receipt of your report within 48 hours.
- We will provide an estimated timeline for the fix.
- You will be notified when a fix is available and released.
- We will appreciate your contribution and may credit you (with your permission) in our release notes.
