# Project Governance

This document outlines the governance model for Mradul Sharma's Portfolio repository.

## 1. Roles and Responsibilities
- **Maintainer / Owner (Mradul Sharma):** Responsible for managing the project direction, reviewing major pull requests, merging community contributions, and performing production releases.
- **Contributors:** Anyone who submits a bug report, feature request, or pull request to improve the project.

## 2. Decision Making Process
Major architectural decisions or changes to the tech stack are evaluated based on community issues and pull request discussions. The Maintainer retains the final say on merging changes to ensure stability, code quality, and alignment with the portfolio's primary goal.

## 3. Code Reviews
All code changes must be submitted via Pull Requests. Direct commits to the `main` or production branch are prohibited. Pull Requests require:
- A clear description of the change.
- Linting checks to pass.
- Approval from at least one core maintainer before merging.

## 4. Conflict Resolution
In the event of disputes or disagreements over code directions, participants are expected to refer to the `CODE_OF_CONDUCT.md` and keep conversations respectful. The Maintainer will provide the final resolution.
